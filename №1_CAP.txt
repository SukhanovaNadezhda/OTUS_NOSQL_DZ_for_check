Задание:Необходимо написать к каким системам по CAP теореме относятся перечисленные БД и почему:
MongoDB, MSSQL, Cassandra

Решение: 
1.MongoDB: CP
2.MSSQL: CA
3.Cassandra: AP

Выполненный анализ:
1.MongoDB: CP (consistency, partition tolerance): при наличии проблем со связью между узлами, при формировании запроса на чтение или обновлении данных будет возвращаться ошибка:
	- MongoDB  —  это одномастерная структура (и все записи идут по умолчанию в него).
      - Когда первичный узел вне доступа, то вторичный узел выбирается в качестве нового первичного узла.
      - Поскольку клиенты не отправляют запросы на запись в течение интервала смены ролей, данные остаются согласованными по всей сети.
2.MSSQL: CA
      - В реляционных базах данных все связанные данные хранятся в одном узле или на одной машине. 
3.Cassandra: AP
	- Кластер Cassandra представляет собой набор узлами, соединенных в одноранговой распределенной архитектуре (главного узла нет, и каждый узел может выполнять все операции с базой данных, и каждый может обслуживать запросы клиентов). Все узлы равноценны.
      - Данные распределяются по узлам на основе согласованного хэша их ключа разделения.
      - Данные имеют коэффициент репликации, который определяет количество копий (реплик), которые должны быть сделаны. Реплики автоматически сохраняются на разных узлах.
      - Узел, который первым получает запрос от клиента, является координатором. В обязанности координатора входит пересылка запроса узлам, содержащим данные для этого запроса, и отправка результатов обратно координатору. 
      - Любой узел в кластере может выступать в качестве координатора.
      //То есть у Cassandra нет лидера, можно обращаться к любому узлу кассандры, и будет получен ответ. Это позволяет cassandra быть высокодоступной, не имея единой точки отказа. Cassandra хранит данные равномерно распрделяя их по своему класстеру узлов. Каждый узел отвечает за часть данных
